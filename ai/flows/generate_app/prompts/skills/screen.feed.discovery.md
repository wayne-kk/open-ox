# Screen Skill: Feed Discovery

Use this skill when the app's primary workflow is discovery, browsing, and lightweight interaction.

## Screen Intent

- Keep one dominant feed surface as the center of the page.
- Preserve continuity: users should be able to scroll, scan, and act without context breaks.

## Recommended Regions

1. Top context row (title, quick filter, mode switch).
2. Primary feed stream (card-based, image-forward when relevant).
3. Lightweight action lane (compose, save, follow, or filter shortcuts).
4. Compact feedback cues (counts, active state, or recent interaction result).

## Design Rules

- Mobile-first spacing and tap targets.
- Two-column feed is allowed if text remains readable; otherwise use one column.
- Keep card metadata concise (author/topic/time/like).
- Avoid long paragraphs and marketing narratives.
