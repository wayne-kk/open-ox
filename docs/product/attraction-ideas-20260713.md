### 1.1 与体验想法池的边界

| 文档 | 焦点 |

### 1.2 产品锚点

- **定位**：从一句自然语言到可运行、可验证、可迭代的真实 Next.js 站点。
- **可被放大的差异化**：真实工程可导出、透明多 Agent 管线、Design Mode 源码写回、Community / Remix、BYO Vercel、飞书等已有集成。
- **吸引力原则**：欲望要具体；传播要可截图/可切片；叙事要站得住（尤其合法与「非抄袭」边界）。

### 1.3 吸引力口径（1–5）

会不会让人「想试一把 / 想转发给别人 / 觉得只有你们能做」。5 = 欲望明确且易传播。

## 2. 获客钩子

### 2.1 竞品 URL → 你的品牌站（「拆了重装」）

粘一个竞品或偶像站 URL → 抽版式 / 模块 / 文案骨架 → 换成用户品牌与色板 → 生成真实 Next.js 工程。

| | |
|--|--|
| **吸引力** | **5** |
| **爽点** | 欲望极明确：「我要那种，但是我的」；比空白 prompt 转化高 |
| **Open-OX 杠杆** | Playwright / 参考能力已有基础；交付物是可导出工程，不是仿站截图 |
| **风险** | 法律与伦理须产品化声明：结构启发 ≠ 抄代码 / 盗图 |
| **建议优先级** | **P0（拉新主轴候选）** |

### 2.2 三版气质分叉，只深造选中的一版

同一 Brief 先便宜出 3 个「气质样张」，用户点选后再烧完整生成管线。

| | |
|--|--|
| **吸引力** | **5** |
| **爽点** | 决策像选皮肤；社交上天然适合「我选了 B」截图 |
| **与体验池蓝图的区别** | 蓝图管 IA / 模块；这里管**视觉方向分叉**——解决另一半「气质不对」的赌博 |
| **可组合** | 与 [`generate-blueprint-preview-v0.1-prd.md`](./generate-blueprint-preview-v0.1-prd.md) 组合成「结构确认 × 气质确认」 |
| **建议优先级** | **P0（拉新主轴候选）** |

#### 展示形态（讨论结论草案 · 2026-07-13）

**结论：不要只丢三张「假成品」大图。** v0.1 推荐 **「气质卡片 = Token 实装小样 + 一句标签」**；图生静帧最多作辅，不当唯一真相。

| 形态 | 是什么 | 优点 | 风险 | 建议 |
|------|--------|------|------|------|
| A. 纯 AI 静帧大图 | 文生图出一张 Hero/整页假站 | 最「好看」、传播强 | 与最终 Next.js **必然漂移**；用户以为选的是成品 | 可作辅视觉，不作唯一决策面 |
| B. 设计系统卡片 | 色板 + 字阶 + 圆角/间距 + mood 词 | 便宜、诚实、直接进 `designIntent` | 「不够像网站」，爽感弱 | 必做底座 |
| C. **Token 实装小样（推荐）** | 同一极简壳（固定 Hero 骨架），三套 token 渲成三张可点预览 | 比色板更像站，又比真预览便宜；和管线产物同构 | 要做一个极简壳组件 | **v0.1 主形态** |
| D. 真跑三条迷你预览 | 三条缩短管线出可点 URL | 最真 | 贵、慢，违背「先便宜再深造」 | 不做 |

**推荐用户看到的一张「样张」长这样：**

1. **主视觉**：固定版式的「风格海报」小预览（非完整站点）——同一 Brief 标题/一句副文案，套该方向的色/字/按钮/背景气质。  
2. **侧栏/脚注**：3–5 个色点、主字体名、2–3 个 mood 词（如「克制 · 编辑感 · 大量留白」）。  
3. **文案锚**：一句话差异（「A 冷淡科技 / B 温暖人文 / C 大胆促销」），降低「三张看起来差不多」的选择疲劳。  
4. **心理预期文案**：明确写「这是气质方向，不是成品预览；结构与模块仍以蓝图/生成结果为准」。

**为何不主推「整页截图」：**  
蓝图 PRD 已要求线框刻意低保真，避免与成品混淆。气质样张若做成高保真假站图，会制造相反的信任坑——选了好看图、生成出来「不像」，比不选更伤。Token 实装小样把决策钉在**可进入管线的 design tokens / designIntent** 上，而不是钉在一张不可复现的渲染图上。

**若仍用「纯 AI 静帧」作辅视觉，图从哪来（现有能力）：**

```text
Brief
  → LLM 产出 3 条 designDirection（mood / 色 / 字 / 禁止项 + 英文出图 prompt）
  → 并行调用已有 Ark 文生图（lib/ark-image-generate.ts → generateArkImageBase64）
  → 得到 3 张 JPEG/PNG（约 1k），存临时对象或 data URL
  → 卡片上展示；用户点选后，写入管线的仍是该方向的 designIntent/tokens，不是像素
```

| 项 | 说明 |
|----|------|
| **引擎** | 火山方舟 Ark `/images/generations`（现网默认模型 `doubao-seedream-*`，见 `ARK_IMAGE_MODEL`） |
| **调用面** | 与 Modify/生成里的 `generate_image` tool 同源（`ai/tools/system/generateImageTool.ts`） |
| **Prompt** | 宜短英文（Ark 侧约 160 字截断）；写「landing hero mood board / style frame」，**禁止**要求可读 UI 文案、logo、真控件 |
| **出图内容** | 氛围板 / 局部 Hero 摄影质感 / 材质与光影——**不是**可点击的假网页截图 |
| **耗时与成本** | 单张大约数秒～十几秒级 + 按次计费；三张可并行，仍比完整生成管线便宜，但比纯 Token 小样贵一个数量级 |
| **失败降级** | `ARK_API_KEY` 未配或出图失败 → 只展示 Token 小样 + 文案标签，不挡确认门 |
| **不要用的路径** | Playwright **封面截图**（`cover/capture`）——那是真站点渲染后的截图，气质分叉时尚无站可截 |

和蓝图怎么叠：先确认结构（线框）或先确认气质（小样）皆可；若同屏，气质用「皮肤选择器」心智，蓝图用「模块编辑器」心智，不要合成一张「半真半假的整页」。

#### 怎么展示、选完会发生什么（交互草案）

**出现时机（插在 Intent 确认层）**

```text
用户一句话
  →（可选）clarify / 三问
  → confirm_brief（散文 Brief 已对齐）
  → ★ confirm_vibe：三选一气质样张   ← 本节
  →（可选）confirm_outline：蓝图模块
  → 用户点「开始生成」→ commit_generate
  → 完整管线只按「选中的那一版」深造
```

**屏幕上长什么样**

- Studio 对话流里出现一块 **「选一个气质」** 面板（不是跳走新页）。
- **三张卡片横排**（手机可横滑）；单选，点一张即高亮描边 + ✓，可改选，未确认前不烧完整管线。
- 每张卡片自上而下大致是：
  1. 主视觉：Token 实装小样（主）± Ark 氛围图作顶图/背景（辅）
  2. 标题一句：「A · 冷淡科技」
  3. 色点 + mood 词
  4. 极短说明（「深色、细线、少装饰」）
- 面板底：一句预期文案 + 主按钮 **「用这个气质生成」**（未选中时 disabled）。
- 次要操作：**「都不喜欢，再说一个方向」**（重抽，计少量成本）/ **「跳过，用默认」**（走 Brief 里已有风格句，不强制三选）。

**选中当下（点卡片，还没点生成）**

| 效果 | 说明 |
|------|------|
| UI | 仅该卡选中态；另外两张变淡，不删除，方便改选 |
| 会话 | 对话里可落一条短确认：「已选气质：冷淡科技」 |
| 数据 | 前端暂存 `selectedVibeId` + 该方向的 `designIntent` 草稿；**尚不 enqueue 生成** |
| 费用 | 出样张时已花的便宜成本（LLM+可选三图）；完整 Credits 仍未扣 |

**点「用这个气质生成」之后**

| 效果 | 说明 |
|------|------|
| 锁定 | 选中方向写入 payload（建议字段：`confirmedDesignDirection` → 进入 `experience.designIntent` / 设计系统约束） |
| 丢弃 | 另外两版只作会话展示，**不进** analyze / design system / page implement |
| 启动 | 走现有 `commit_generate` → 完整生成管线（与今天点开始生成相同，只是风格约束更硬） |
| 用户感知 | 进入熟悉的 Atlas / 步骤进度；可选在状态条显示「气质：冷淡科技」 |
| 成品预期 | 色板、字体、疏密、装饰密度应贴所选方向；**版式模块仍由蓝图/Brief 决定**，不会因为选了「暖色」就多出 Pricing |
| 不允许 | 静默用未选中的另外两版「再赌一把」；也不把氛围图当 Hero 真图强制塞进站点（除非后续管线自己按 designIntent 再 `generate_image`） |

**选完不会发生的事（避免期望错位）**

- 不会立刻出现可点击的完整网站预览（那是生成结束后的事）。
- 不会保证成品和 Ark 氛围图像素级一致。
- 不会并行跑三条完整生成（那是贵且慢的反模式）。

**和「只确认 Brief」比，多出来的用户价值**：用户用眼睛做了一次低成本决策；后面那一次贵生成，风格赌博被收窄成「三选一里的那一个」。

## 3. 传播与立场

### 3.1 造站直播间（管线即演出）

生成过程可一键开「公开观摩」：多 Agent 进度、关键决策、最终预览；提供短视频 / B 站切片模板。

| | |
|--|--|
| **吸引力** | **4.5** |
| **爽点** | 透明管线从工程优势变成**可传播内容**；创作者 / 讲师可带流量 |
| **Open-OX 杠杆** | Generation Atlas / build steps 已有；差的是可分享剧场与外链形态 |
| **建议优先级** | P1（内容增长） |

### 3.2 「反 AI 味」体检官 + 一键祛味

打分：紫渐变、圆角胶囊、默认无衬线栈、假数据、空洞 slogan 等；给「祛味」Modify 一键。

| | |
|--|--|
| **吸引力** | **4.5** |
| **爽点** | 把「讨厌 AI 站同质化」产品化；分数天然可分享 |
| **叙事** | 「不是又一个 AI 建站，是专门治 AI 建站病的」 |
| **落地策略** | 可先评分、后自动修；与设计品味 skill 同向 |
| **建议优先级** | **P0（叙事 / 传播主轴候选）** |

## 4. 信任与生态

### 4.1 一键打开本地 IDE / Cursor 互通

Studio 提供「在 Cursor / VS Code 打开」；本地改完可同步回预览；ZIP 从终点变成默认习惯的一环。

| | |
|--|--|
| **吸引力** | **4**（开发者与「要掌控感」人群） |
| **爽点** | 打穿「AI 建站锁死平台」恐惧；兑现「真实工程可带走」 |
| **传播点** | 开发者社区友好，易形成「这才是正经的」口碑 |
| **建议优先级** | P1（口碑楔子） |

### 4.2 Skill / 气质包市场

创作者上架「日式极简 / 咨询顾问 / 电商品牌」等 Skill 包；生成时选用，付费或 Remix 抽成。

| | |
|--|--|
| **吸引力** | **4** |
| **爽点** | 双边：用的人图省事，做的人图分成；Community 从逛站变成逛「可安装审美」 |
| **Open-OX 杠杆** | `public/skills/` 已有用户向 skill 包雏形 |
| **建议优先级** | P2（生态护城河；宜在 Community 稳定后） |

### 4.3 外包接手：导入 ZIP / GitHub 再改

不是从 0 生成；导入已有站点工程 → 理解结构 → 用 Modify / Design Mode 接着改。

| | |
|--|--|
| **吸引力** | **4** |
| **爽点** | 「我已有半成品」是真实大市场；竞品多假设 blank canvas |
| **定位切换** | 从生成器变成 **AI 外包接力棒** |
| **建议优先级** | P1（抢半成品市场） |

## 5. 明确付费 / 运营场景

### 5.1 活动页 48h 限时站

选模板 → 填活动信息 → 生成 → 自动 Deploy + 到期下线 / 归档。

| | |
|--|--|
| **吸引力** | **3.5–4** |
| **爽点** | 场景极具体（闪购、报名、线下活动）；决策成本低，易复购 |
| **生意** | 短生命周期高频 → Credits 自然消耗 |
| **建议优先级** | P2 |

### 5.2 客户提案包一键导出

预览链接 + 一页 PDF（目标用户 / 模块 / 气质 / 报价占位）+「批准后生成完整站」。

| | |
|--|--|
| **吸引力** | **3.5** |
| **爽点** | 自由职业者见客户的武器；靠他们二次分发获客 |
| **与体验池「客户批注」的区别** | 批注是改站协作；提案包是**成交前销售件** |
| **建议优先级** | P2 |

### 5.3 同题竞技场（每日 Brief）

官方每日同一 prompt，用户生成后可投稿 Community「今日题」榜。

| | |
|--|--|
| **吸引力** | **3.5** |
| **爽点** | 游戏化回访 + 内容供给；低成本运营拉新 |
| **注意** | 须控 Credits，避免烧钱比赛 |
| **建议优先级** | P2 |

### 5.4 站群战役工厂（同品牌 N 个落地页）

锁定设计系统，批量生成「渠道 A/B / 城市站」变体。

| | |
|--|--|
| **吸引力** | **3.5** |
| **爽点** | 投流 / 本地推刚需；客单价高于单页玩票 |
| **与「多页站点」的区别** | 不是信息架构变深，而是**同系统横向复制** |
| **建议优先级** | P2（偏商业化场景） |
