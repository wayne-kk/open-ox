## 一、项目概况

### 1.1 项目背景

在跨端 Web / 移动 Web 产品交付中，界面从需求描述到高保真落地往往依赖多角色手工协作：信息架构、设计系统、全局导航壳层、分区块组件、动效与主题变量分头实现，易出现风格漂移、组件重复与预览不可构建等问题。通用 AI 聊天工具可输出零散 TSX/CSS，但难以保证**跨页面一致的设计语言**、**可运行的 Next.js/React 工程**以及**生成后的可视化迭代**。

本项目立项建设《跨端前端界面智能生成与美化渲染系统 v1.0.0》（工程实现 **Open-OX Studio**），围绕「自然语言 → 页面蓝图 → 设计系统与 Design Tokens → 全局 Chrome 脚手架 → 分页面 Section 实现与美化 → 构建验证 → 跨端可预览交付」形成闭环，将前端界面生成与美化渲染固化为可复用系统能力。

### 1.2 建设目标

1. **跨端前端界面智能生成**：支持 Web（Next.js App Router）站点多页/单页蓝图，自动产出 `layout.tsx`、导航/页脚等 chrome 及各页 Section 组件。
2. **美化渲染与设计系统落地**：内置 design-system / hero / section 技能库，将 Tailwind v4 CSS 变量、动效（Framer Motion / GSAP / WebGL Hero）规范写入 `globals.css` 与组件。
3. **高保真预览与迭代**：Studio 控制台 SSE 实时进度、Storage / E2B / 本地多模式预览；生成后自然语言修改 UI（搜索 → 编辑 → 构建门控）。
4. **工程级质量保障**：scoped TypeScript 检查、`next build` 与最多 5 轮 `repair_build` 自愈，交付真实可运行前端工程而非静态截图。

### 1.3 完成情况概述

V1.0.0 已完成核心研发：主栈 **Next.js 16 + React + TypeScript + Tailwind CSS v4**；生成流水线 `ai/flows/generate_project`（意向对话、需求分析、设计意图、页面规划、设计系统匹配、Token 注入、Architect、Page Implement、生图与依赖、构建修复）；修改流水线 `ai/flows/modify_project`。典型单页落地页端到端约 **90 秒**；页面级 Agent 可 **7+ 路并行**；用户零手写代码即可完成生成与预览。

### 2.2 系统总体架构

```
用户浏览器（首页 HeroPrompt / Studio）
        │ 自然语言 + 可选参考截图 / 风格技能
        ▼
POST /api/projects → POST /api/ai（SSE 推送生成步骤）
        │
        ├── 前端蓝图与设计系统（Blueprint、design-system.md、globals.css）
        ├── Architect Agent（跨页 chrome 脚手架）
        ├── Page Implement Agent × N（各页 Section + 美化 skill）
        ├── await_images（界面配图异步落盘）
        └── 预览渲染（Storage / E2B / local）→ 用户可见跨端 Web 界面
```

## 三、关键技术创新点

### 3.1 设计系统驱动的跨端一致性

- **Single Source of Truth**：`design-system.md` + `globals.css` CSS 变量为全站渲染真源，下游 Page Agent 禁止擅自发明品牌色与字体体系。
- **技能化美化**：`prompts/skills/design-system/` 与 `section/hero/` 将高阶视觉效果（WebGL Hero、滚动动效、玻璃拟态等）沉淀为可复用模板，按页路由注入。

### 3.2 「先壳层、后页面」的界面编排

- Architect 固定 `app/layout.tsx` 与 `components/chrome/**`，Page Agent 在只读 layout 契约下编写 `{scope}_*Section.tsx`，避免导航/footer 重复渲染。
- `knownRoutesPrompt` 约束多顶层路由（`home`→`/`）下链接与导航一致性，支撑跨页面跳转体验。

### 3.3 并行 Section 生成与渲染资源协同

- 多页面 `page_implement_agent` Promise 池并行；`await_images` 与 `install_dependencies` 与构建前并行，保证界面图片与 npm 组件库就绪后再 `next build`。

### 3.4 美化渲染规则束（Guardrail Bundles）

- `agentRuleBundles` 按序加载 `tailwindMappingGuide`、`outputTsx`、`skillIntegrationContract`、`framerMotionVariants` 等，将 LLM 输出约束在 Tailwind 语义类与可编译 TSX 内。

### 3.5 生成后界面迭代与可视 diff

- 用户以自然语言描述「导航颜色」「Hero 标题大小」等视觉问题；Agent 先 `search_code` 定位 Section 文件，再 `edit_file` 精确替换，**Stop Hook** 强制 `run_build` 通过后才结束。

### 3.6 跨环境预览渲染

- **Storage**：静态导出 + 同源 `/site-previews` 代理，浏览器直接渲染完整 Web UI。
- **E2B**：云端 `next build` + `serve out`，适合演示与验收。
